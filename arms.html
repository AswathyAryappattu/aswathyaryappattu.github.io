Project 003

ARMS — Assisted Robot for Medical Suite

A robotic arm designed for contagious isolation wards that monitors patient vitals, automates IV infusion management, and enables remote monitoring — reducing direct contact between healthcare workers and patients.

ESP32 Arduino IDE Embedded C Firebase React IoT RFID MAX30102 MLX90614 Servo Motors
Status Completed
Institution GEC Palakkad
Team 4 Members
Domain Medical Robotics

Overview

In contagious wards, frequent patient monitoring increases the risk of spreading infections among healthcare workers. ARMS addresses this by introducing a robotic arm that measures vital signs — heart rate, SpO2, and body temperature — in real time using advanced sensors, without requiring a nurse to be physically present at the bedside.

The system also tracks IV fluid levels and automatically alerts staff when a drip runs out, preventing complications. Patients are identified using RFID tags, and all health data is stored in Firebase and accessible through a web-based dashboard for remote monitoring.

This project was developed at Government Engineering College, Palakkad, and is especially relevant to high-risk environments like COVID-19 isolation wards where minimizing human contact is critical.

Hardware Components

Component Function
ESP32-WROOM Central microcontroller — handles sensors, actuators and wireless communication
MAX30102 Heart rate and SpO2 (blood oxygen) monitoring
MLX90614 Non-contact infrared body temperature measurement
RFID-RC522 Patient identification via RFID tags
Servo Motors Robotic arm movement — rotation, lifting, and gripping
Solenoid Valve IV fluid flow regulation
Relay Module Automated control of infusion management

Software Stack

Firmware

ESP32 programmed using Arduino IDE and Embedded C. Handles real-time sensor reading, servo control, solenoid valve actuation, and RFID scanning.

Cloud & Dashboard

Firebase Realtime Database stores all sensor data and IV details. A web dashboard built with React and CSS lets nurses monitor vitals and IV status remotely with secure login via Firebase SDK.

Key Features

  • Real-time monitoring of heart rate, SpO2 and body temperature without physical contact
  • Automated IV fluid level tracking with nurse alerts when drip runs out
  • RFID-based patient identification for accurate record-keeping and treatment plans
  • Robotic arm with servo motors for flexible sensor placement (rotation, lifting, gripping)
  • Firebase cloud storage for continuous data updates and remote access
  • Secure web dashboard with login authentication for authorized medical staff only
  • Reduces infection risk by minimizing direct contact between nurses and patients

Results

The ARMS system was successfully implemented and tested, with all hardware and software components integrated and functional.

3 Vitals Monitored in Real Time
✓ Automated IV Alert System
✓ Remote Web Dashboard Live

Challenges & Learnings

Integrating multiple I2C sensors (MAX30102 and MLX90614) on the same bus required careful address management and timing to avoid data collisions. The servo arm positioning also needed precise calibration to ensure sensors made reliable contact with the patient.

Working with Firebase real-time sync taught the team how to structure cloud data efficiently to minimize latency in the dashboard updates. Implementing secure authentication via Firebase SDK ensured patient privacy was maintained throughout.

Project 001

Autonomous Navigation Robot

A ROS1-based mobile robot that navigates unknown environments autonomously using SLAM, real-time obstacle avoidance, and a custom path planning node.

ROS1 C++ SLAM LiDAR Python Gazebo
Status Completed
Duration 3 months
Role Solo Project
Year 2024
🤖

Overview

This project involved building a fully autonomous mobile robot capable of navigating unknown environments without any prior map knowledge. The robot uses a 2D LiDAR sensor to perceive its surroundings and constructs a map in real time using the GMapping SLAM algorithm.

All computation runs on a ROS1 Noetic stack. A custom C++ node handles the path planning logic using A* search on the occupancy grid published by the SLAM module, while a separate Python node manages obstacle avoidance with reactive control.

Key Features

  • Real-time SLAM using GMapping with 2D LiDAR for simultaneous localization and mapping
  • Custom A* path planner implemented as a ROS1 C++ node, subscribing to the occupancy grid
  • Reactive obstacle avoidance layer using potential fields to handle dynamic obstacles
  • Simulation tested in Gazebo before physical deployment on a differential drive robot
  • Modular ROS1 architecture with separate nodes for perception, planning, and control

Results

The robot was tested across 5 different environments. Key performance metrics from the final evaluation:

92% Navigation Success Rate
<5cm Localization Error
0 Collisions in Testing

Challenges & Learnings

One of the main challenges was tuning the costmap inflation radius to balance between safe clearance from obstacles and allowing the robot to navigate narrow corridors. Too large an inflation value caused the planner to fail in tight spaces.

Another challenge was latency between the LiDAR scan and the costmap update — solved by optimizing the ROS node update rates and reducing unnecessary message publishing frequency.
